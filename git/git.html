Set Remote Repository

    • Git configuration
    Set Global configuration
  • git config --global user.name "FIRST_NAME LAST_NAME"
  • git config --global user.email "MY_NAME@example.com"
    • View global configuration
  • vi ~/.gitconfig
    • Set Repository specific configuration (execute commands from repository directory)
  • git config user.name "FIRST_NAME LAST_NAME"
  • git config user.email "MY_NAME@example.com"
    • View repository specific configuration
  • git config -l
    • Branch commands
    Command to create branch and does not switch to the created branch, but remain in the current branch.
  • git branch <Branch Name>
    • Create branch and switch to the created branch
  • git checkout -b <Branch Name> <From branch name>
    • Switch to the previous branch
  • git checkout -
    • Push a branch to remote repository
  • git push --set-upstream origin <New Branch Name>
    • OR
  • git push -u origin <New Branch Name>
    • Push first time after setting remote URL
    1. When there are no earlier commits (Not added .gitignore / Readme.md files in remote repository)
  • git remote add origin
  • git pull origin master
  • git push -u origin master OR git push --set-upstream origin master
    • 2. When there are some earlier commits
  • git remote add origin
  • git pull origin master --allow-unrelated-histories
  • git push -u origin master OR
  • push --set-upstream origin master

    Cherry pick - Picking any specific commit from other branch and pull same changes in your branch (committed/added)
    Pick commit hash from other branch and use in your branch to get all the changes from other branch to this branch, it will be committed into your branch as well.
  • git cherry-pick <commitHash>
    • Same as above but will not commit, you can change and provide a custom commit message and commit again
  • git cherry-pick <commitHash> -n
    • Fetch branch(es) from remote repository to your local - Syncs all the branches from remote to your local. (Branch which is created by other developer or in other workspace)
  • git fetch origin
    • Fetch any specific branch
  • git fetch origin <branchName>
    • Cherry-pick: pulling any specific commit from other branch to your branch. Pick changes and commit into current branch
  • git cherry-pick <commit Hash>
    • Pick changes and don't commit automatically, Now make your changes and commit.
  • git cherry-pick <commit Hash> -n
    • Git Add/Remove - Add all the files in present directory and its sub-directories
  • git add .
    • Add all the files in present directory, its sub directories and also from parent directories
  • git add -A
    • Add a specific file(s) or directory
  • git add <file1 With Path> <file2 With Path>
    • Remove added file(s) / Unstage multiple files
  • git rm --cached <File Name 1> <File Name 2>
    • To unstage multiple files recursively
  • git rm -r --cached <Directory Name> /.
    • Convert a added file to non added file [stage to unstage]
  • git restore --staged <file Name>
    • Now if you want to remove changes from unstage
  • git restore <file Name>
    • Git Commit : Committing added files to local repository - Commit all added files
  • git commit -m "Message"
    • Update commit message - Will update last commit message.
  • git commit --amend -m "New Message"
    • Add new changes into previous commit
  • git commit --amend
    • Display all files which are committed in this.
  • git show <Commit hash>
    • Remove last commit
  • git reset --hard HEAD^
    • Push above deleted commit into remote repository
  • git push -f
    • OR
  • git push --force
    • After reset changes will be present in working directory
  • git reset --soft <Commit Hash>
